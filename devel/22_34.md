# 22_34: 入门第三个任务：above和below不应该被翻译

## 2025/10/17
### What
让 `\above`/`\below` 作为宏命令在“状态栏/焦点条”中保持英文显示；同时保留菜单、气泡提示等 UI 文本的中文翻译。

### Why
在TeXmacs的宏命令系统中：
- `\above` 和 `\below` 是技术性的宏命令
- 类似于编程语言中的if/else等关键字
- 被翻译成"以上"和"下面"后，破坏了技术术语的一致性
- 影响用户对宏命令系统的理解和使用

### 如何测试
1) 在编辑区输入 `\` 进入命令模式，键入 `above` 回车：底部状态栏/焦点条应显示英文 `above`（不翻译）。
2) 同上输入 `below`：状态栏/焦点条显示英文 `below`（不翻译）。
3) 打开相关菜单（如“数学→脚本/标记”等）：菜单项、图标气泡仍显示中文（例如“顶标/底标”、“在上边插入/在下边插入”）。

### 技术实现
- 不再删除翻译词条。`TeXmacs/plugins/lang/dic/en_US/zh_CN.scm` 中的 `("above" "以上")`、`("below" "下面")` 保留，以保证菜单与工具提示继续中文化。
- 仅在“焦点条渲染宏名”的位置施加 `verbatim`，使宏名不被翻译：
  - 文件：`TeXmacs/progs/generic/generic-menu.scm`
  - 变更：在 `focus-tag-icons` 渲染处的两处名称输出，包一层 `(eval \`(verbatim ,(focus-tag-name (tree-label t))))`：
    - 当只有一个变体时的 `inert` 标签名；
    - 有多个变体时 `balloon` 的标题标签名。
- 同时恢复 `focus-insert-menu` 中 “Insert above/Insert below” 为普通字符串，让其仍走翻译系统（菜单显示中文）。

这样实现“宏命令英文 + UI 中文”的双目标：宏名由 `verbatim` 直接输出，绕过翻译；普通 UI 文本继续经由翻译表。
