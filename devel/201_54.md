# [201_54] 修复 CJK 文本下 Ctrl+左右按词移动异常

## 如何测试

1. 打开包含中文/日文/韩文（CJK）文本的文档，混合中文标点与空格
2. 将光标置于文本中间，反复按 Ctrl+Left / Ctrl+Right
- **macOS** 下对应快捷键为 **Command (⌘) + Left / Right**
3. 预期行为：
   - 光标应在“词 / 标点”边界处停住
   - 不应出现一次跳过多个汉字或结构节点的情况
4. 对比英文文本，Ctrl+Left / Ctrl+Right 行为应保持不变

### （示例）

**测试文本：** 你好，世界。ABC def


**测试步骤：**
1. 将光标放置在如下位置：你|好，世界。ABC def
2. 按三次 **Ctrl + Right**
3. 光标会依次出现在：
- 你好|，世界。ABC def   
- 你好，世界|。 ABC def
- 你好，世界。 ABC| def
4. 再按三次**Ctrl + Left**
5. 光标会依次出现在：
- 你好，世界。|ABC def   
- 你好，|世界。 ABC def
- |你好，世界。 ABC def

## 2026/1/15
### What 
- 修复在 CJK 文本中使用 Ctrl+Left / Ctrl+Right 时，光标按词移动行为异常的问题
- 调整分隔符（word separator）判断逻辑，使中文与全角标点被正确识别

### Why 
- 现有实现主要依赖 ASCII 字符判断，导致在中文等 CJK 文本中：
  - 中文标点无法作为有效的词边界
  - Ctrl+左右键表现为结构级跳转而非按词移动
- 该问题已由用户在问题报告中反馈，影响编辑体验

### How 
- 在 `tree_traverse.cpp` 中补充分隔符判定逻辑
- 将 CJK 字符及全角标点纳入 word separator 范畴
- 保持原有的结构遍历逻辑，仅增强“词边界识别”部分
